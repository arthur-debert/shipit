`shipit release sign` is the consumer-agnostic macOS signer unit. The model
is bundle(unsigned) → sign-reopens-and-reseals, never sign → bundle
(workflows.lex §3.1). The unit carries TWO legs, dispatched by what the
bundle tree carries:

  mac-app leg     the tree carries an unsigned .app reseal payload
                  (+ at most one .dmg) — the mac-app composition's shape
  archive leg     the tree carries plain .tar.gz archive bundles — the
                  archive composition's darwin raw-binary shape

The mac-app leg, end to end:

1. unpack the unsigned .app from its reseal payload
   (<name>.unsigned-app.tar.gz — a tarball because artifact upload destroys
   a .app's symlinks and exec bits). Exactly one .app and at most one .dmg;
   zero or multiple is a hard error, never a nondeterministic pick;
2. enumerate the nested signable Mach-O inside the .app (extra executables,
   loose dylibs, helper bundles — recursively; frameworks are opaque and
   signed at their root);
3. codesign inner-first with the .app LAST — hardened runtime + secure
   timestamp on every Mach-O (a flat sign leaves nested code unhardened and
   the notary rejects it). Entitlements are shipit-provided and keyed on the
   bundle SHAPE (there is no flag): an electron bundle — detected by its
   Electron Framework — gets the JIT entitlements pair Chromium/V8 needs to
   run under hardened runtime (allow-jit on the top .app, allow-jit + inherit
   on each helper .app), applied PER CODE ROLE; a mac-app/tauri/rust .app and
   a nested .appex never receive JIT. The Developer ID .p12 is imported into a
   per-invocation temporary keychain torn down on every exit, with unique
   keychain/cert paths per pass so the .app and .dmg passes cannot collide;
4. reseal the .dmg FROM the signed .app via hdiutil (re-bundling would strip
   the signature), then codesign the resealed .dmg;
5. notarytool submit / poll / staple against the signed .dmg, then stage it
   under the original dmg filename (--out, default: TREE itself).

The archive leg (the legacy rust-cli.yml sign steps' shipit equivalent):

1. reopen every plain .tar.gz archive bundle (the loose staging binary
   beside it loses its exec bit in artifact transport; the tar's headers
   preserve it) and find the shipped binaries by CONTENT — Mach-O magic,
   never filename;
2. codesign every binary across all archives in ONE temporary-keychain pass
   — hardened runtime + secure timestamp, verify --strict per path;
3. notarize each signed binary: zip it (notarytool needs a container),
   submit, poll. NO staple — a bare binary has no staple target; Gatekeeper
   verifies the notarization online;
4. re-emit each tarball from its signed tree and stage it under the
   original archive filename (--out, default: TREE itself).
   Entitlements never reach this leg: they are the mac-app leg's
   per-code-role policy over an unpacked .app, and a raw CLI binary carries
   none (legacy rust-cli parity).

The unit makes zero tauri or electron assumptions: whatever bundler produced
the unsigned input is the caller's business. It needs no git checkout and no
.shipit.toml — it runs anywhere a mac, the tree, and the credentials are,
laptop or CI. The assert-bundle integrity guard at the signer's entry is the
wf-sign-mac workflow block's job, not this unit's.

Credentials, from the environment (both legs, identically):

  signing         APPLE_CERTIFICATE (base64 Developer ID .p12) +
                  APPLE_CERTIFICATE_PASSWORD (an EMPTY password is valid —
                  passwordless .p12 is legal PKCS#12)
  notarization    either the ASC API key trio (ASC_API_KEY_BASE64 /
                  ASC_API_KEY_ID / ASC_API_ISSUER_ID — wins when both styles
                  are present) or the Apple-ID trio (APPLE_ID /
                  APPLE_PASSWORD / APPLE_TEAM_ID)

Missing signing or notary secrets is a HARD FAIL naming the missing names —
the legacy warn-and-skip is gone. Missing-secrets detection belongs to
preflight; the only unsigned path is the explicit --unsigned break-glass
decided upstream, never an ambient skip inside the signer. Decoded credential
material (.p12/.p8) is wiped on any exit, success or failure.

This unit is act-untestable — codesign/notarytool need a real macOS runner
and real Apple credentials — so its verification is remote-only: the
TOL02-WS07 lex rc cut (mac-app routing), the full .app/.dmg leg with
phos-app in ADP02, and a signing consumer rc for the archive leg (TOL02-WS08).

Exit codes: 0 signed + notarized (staple failure alone is non-fatal), 1
runtime failure (missing secrets, bad tree, rejected or unconfirmed
notarization), 2 usage.
